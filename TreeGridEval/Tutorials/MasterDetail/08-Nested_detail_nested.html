8. More nested master detail

Master detail relation with deeper tree of detail tables shown as children of master rows

Source: 08-Master.xml   08-Detail.xml   08-DetailDetail.xml
  • The example shows three levels as the master -> detail -> detail relation.
    TreeGrid supports unlimited levels of such relations.
  • The example provides moving and copying rows between and inside detail grids and inside master grid.
    Also copy & paste supports copying and pasting in detail and master grid.
  • All data for all detail grids must be defined in the main master grid, in appropriate tree.
    The example defines tree of rows with Defs: MAIN - > DETAIL -> DETAILROW -> DETAILDETAIL -> DETAILDETAILROW.
    MAIN - data rows in master grid.
    DETAIL - child rows in master grid that show the first detail grid.
    DETAILROW - data rows in the first detail grid.
    DETAILDETAIL - child rows in the first detail grid that show the second detail grid.
    DETAILDETAILROW - data rows in the second detail grid.
